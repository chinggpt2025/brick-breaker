# `/blindspot`｜盲區總結

**用途**：總結「這類問題我為什麼老是看不到」

> 這是唯一會「越用越聰明」的指令

---

## 執行步驟

### 1. 回顧本次對話
找出本次對話中暴露的盲區

### 2. 抽象盲區類型
不是針對具體問題，而是針對「你」的思維模式：

- 這類問題為什麼總是看不到？
- 哪種思維習慣導致了這個盲區？

### 3. 建立預警點
給出下次自動預警點：
- 當出現 [信號] 時，提醒檢查 [盲區類型]

---

## 盲區類型參考

| 類型 | 特徵 |
|------|------|
| 確認偏誤 | 只找支持自己觀點的證據 |
| 可得性偏誤 | 過度依賴容易想到的例子 |
| 錨定效應 | 被第一個資訊固定思路 |
| 過度自信 | 高估自己的判斷準確度 |
| 框架效應 | 被問題呈現方式影響判斷 |
| 倖存者偏差 | 只看到成功案例 |

---

## 輸出格式

```
🕳️ 盲區總結

📍 本次暴露的盲區：
- [盲區描述]

🧠 思維模式診斷：
- 類型：[盲區類型]
- 習慣：[導致盲區的思維習慣]

⚠️ 下次預警點：
- 當 [信號] 出現時
- 提醒檢查：[盲區類型]
```
